# Tick Tack Timer Project Conventions

## Documentation Organization

All project documentation, instructions, and guides should be placed in the `docs/` directory to keep them organized in one central location.

Examples:
- Test instructions
- Deployment guides
- Architecture diagrams
- User guides
- Development workflows

## File Naming

- Use kebab-case for file names: `test-instructions.md`, `deployment-guide.md`
- Use descriptive names that clearly indicate the content

## When Creating Documentation

Always place new documentation files in `docs/` rather than the project root to maintain organization as the project grows.
